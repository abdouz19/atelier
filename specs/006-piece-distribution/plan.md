# Implementation Plan: Piece Distribution Management

**Branch**: `006-piece-distribution` | **Date**: 2026-03-15 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/006-piece-distribution/spec.md`

## Summary

Add two new modules to the Atelier desktop app: **Tailors** (sidebar section for managing sewing contractors, tracking balances and payments) and **Distribution** (sidebar section for distributing cut pieces to tailors and recording returns with stock consumption). On distribute: selected `cutting_pieces` IDs change to `'distributed'` status and a `distribution_batch` record is created linking those exact piece IDs to the tailor. On return: selected piece IDs change to `'returned'`, an optional stock consumption is deducted, and the batch's remaining quantity decreases. All distribution and return records are immutable; tailor payments are editable. Two atomic SQLite transactions handle the distribute and return actions. 6 new tables, 18 IPC channels across 2 namespaces.

**Note on plan input**: The plan input description refers to tailors as "active employees" — this is overridden by the spec clarification (session 2026-03-15) which explicitly established tailors as a separate entity (`tailors` table), independent of `employees`.

## Key Implementation Notes

### Atomic Distribute (`distribution:distribute`)

```javascript
const distribute = db.transaction((payload) => {
  const now = Date.now();
  const batchId = crypto.randomUUID();

  // 1. Validate: tailor exists + active
  // 2. Count available pieces (JOIN cutting_sessions) for model+size+color WHERE status='not_distributed'
  // 3. Validate: payload.quantity <= available
  // 4. SELECT N piece IDs LIMIT payload.quantity
  // 5. INSERT distribution_batches
  // 6. INSERT distribution_piece_links (one per piece ID)
  // 7. UPDATE cutting_pieces SET status='distributed' WHERE id IN (piece_ids)
  return buildTailorSummary(payload.tailorId);
});
```

### Atomic Return (`distribution:return`)

```javascript
const returnPieces = db.transaction((payload) => {
  const now = Date.now();
  const returnId = crypto.randomUUID();

  // 1. Load batch, compute remaining = batch.quantity - SUM(return_records.quantity_returned)
  // 2. Validate: payload.quantityReturned <= remaining
  // 3. Validate: each consumption entry quantity <= available stock
  // 4. INSERT return_records
  // 5. SELECT N piece IDs linked to batch WHERE status='distributed' LIMIT quantityReturned
  // 6. UPDATE cutting_pieces SET status='returned' WHERE id IN (piece_ids)
  // 7. For each consumption: INSERT return_consumption_entries + stock_transactions (consumed, distribution)
  return buildTailorSummary(batchTailorId);
});
```

### Availability Query (for Distribute modal live counter)

```sql
SELECT COUNT(*) AS available
FROM cutting_pieces cp
JOIN cutting_sessions cs ON cs.id = cp.session_id
WHERE cs.model_name = ?
  AND cp.size_label = ?
  AND cs.fabric_color = ?
  AND cp.status = 'not_distributed'
```

### Piece Selection for Distribute

```sql
SELECT cp.id
FROM cutting_pieces cp
JOIN cutting_sessions cs ON cs.id = cp.session_id
WHERE cs.model_name = ?
  AND cp.size_label = ?
  AND cs.fabric_color = ?
  AND cp.status = 'not_distributed'
LIMIT ?
```

### Remaining Quantity for a Batch

```sql
SELECT
  db.quantity - COALESCE(SUM(rr.quantity_returned), 0) AS remaining_quantity
FROM distribution_batches db
LEFT JOIN return_records rr ON rr.batch_id = db.id
WHERE db.id = ?
GROUP BY db.id
```

### Tailor Balance Query

```sql
SELECT
  COALESCE((SELECT SUM(total_cost) FROM distribution_batches WHERE tailor_id = t.id), 0) AS total_earned,
  COALESCE((SELECT SUM(amount) FROM tailor_payments WHERE tailor_id = t.id), 0) AS total_paid
FROM tailors t WHERE t.id = ?
```

### KPI Query

```sql
SELECT
  (SELECT COUNT(*) FROM cutting_pieces WHERE status = 'distributed') AS pieces_in_distribution,
  (SELECT COUNT(*) FROM cutting_pieces WHERE status = 'returned') AS pieces_returned,
  (SELECT COUNT(DISTINCT tailor_id) FROM distribution_batches db
     WHERE (db.quantity - (SELECT COALESCE(SUM(rr.quantity_returned),0)
                           FROM return_records rr WHERE rr.batch_id = db.id)) > 0
  ) AS tailors_with_active_dist,
  (SELECT COALESCE(SUM(total_cost), 0) FROM distribution_batches) AS total_sewing_cost,
  (SELECT COALESCE(SUM(total_cost), 0) FROM distribution_batches)
  - (SELECT COALESCE(SUM(amount), 0) FROM tailor_payments) AS total_unsettled_cost
```
