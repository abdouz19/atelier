# Implementation Plan: Cutting Session Management

**Branch**: `005-cutting-sessions` | **Date**: 2026-03-15 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/005-cutting-sessions/spec.md`

## Summary

Add a Cutting Session Management screen to the Atelier desktop app. The screen shows 5 KPI cards and a sessions table. Users create sessions via a two-step modal: step 1 collects fabric, model, meters, employees, layers, and price; step 2 collects size rows and optional non-fabric stock consumption. On submit, an atomic SQLite transaction deducts fabric stock, deducts consumed items, creates individual piece records (status = not_distributed), and logs earnings to the shared `employee_operations` table. All saved sessions are fully immutable.

## Key Implementation Notes

### Atomic Session Creation (`cutting:create`)

```javascript
const createSession = db.transaction((payload) => {
  const now = Date.now();
  const sessionId = crypto.randomUUID();

  // 1. Validate: fabric availability
  // 2. Validate: each consumption entry availability
  // 3. Validate: all employees exist + active
  // 4. INSERT cutting_sessions
  // 5. INSERT cutting_pieces (loop: pieceCount rows per sizeRow)
  // 6. INSERT cutting_consumption_entries + stock_transactions (consumed, source_module=cutting)
  // 7. INSERT stock_transactions for fabric deduction
  // 8. INSERT employee_operations per employee (type=cutting)
  return buildSessionSummary(sessionId);
});
```

### Fabric Availability Query

```sql
SELECT color,
  SUM(CASE WHEN type = 'inbound' THEN quantity ELSE -quantity END) AS available
FROM stock_transactions
WHERE stock_item_id = ? AND color IS NOT NULL
GROUP BY color
HAVING available > 0
```

### KPI Query

```sql
SELECT
  (SELECT COUNT(*) FROM cutting_sessions) AS total_sessions,
  (SELECT COUNT(*) FROM cutting_pieces) AS total_pieces,
  (SELECT COUNT(*) FROM cutting_pieces WHERE status = 'not_distributed') AS pieces_not_distributed,
  (SELECT COALESCE(SUM(meters_used), 0) FROM cutting_sessions) AS total_meters,
  (SELECT COALESCE(SUM(total_amount), 0) FROM employee_operations WHERE operation_type = 'cutting') AS total_cost
```

### Two-Step Modal State

- `NewCuttingSessionModal` owns `step` (1 | 2) and `step1Data` (validated step-1 values)
- Step 1 → Step 2: `CuttingStep1Form` calls `onNext(step1Values)` → modal saves to state, advances step
- Step 2 Submit: `CuttingStep2Form` calls `onSubmit({ ...step1Data, ...step2Values })` → modal calls `ipcClient.cutting.create()`
- No Zustand; all form state local to the modal subtree

### Pieces Creation

For each `sizeRow`, insert `pieceCount` individual rows into `cutting_pieces`:

```javascript
for (const row of sizeRows) {
  for (let i = 0; i < row.pieceCount; i++) {
    db.prepare('INSERT INTO cutting_pieces ...').run(uuid(), sessionId, row.sizeLabel, 'not_distributed', now, now);
  }
}
```
