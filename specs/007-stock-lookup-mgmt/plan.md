# Implementation Plan: Stock & Suppliers Enhancements

**Branch**: `007-stock-lookup-mgmt` | **Date**: 2026-03-15 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/007-stock-lookup-mgmt/spec.md`

## Summary

Add three managed lookup tables (item_types, colors, units) with predefined entries and user CRUD. Replace free-text type/color/unit inputs across the app with `ManagedDropdown` components sourced from these lists. Add inline-add capability to all color/type/unit dropdowns. Migrate existing free-text data by seeding distinct values. Enforce supplier + price fields as required on every inbound transaction with inline supplier creation.

## Architecture Details

### `ManagedDropdown` Component

Reusable controlled component that replaces simple `<select>` wherever inline-add is needed.

**Props**:
```ts
interface ManagedDropdownProps {
  value: string;
  onChange: (value: string) => void;
  items: LookupEntry[];            // active entries from lookup list
  placeholder?: string;
  addLabel: string;                // e.g. "إضافة نوع"
  onAddNew: (name: string) => Promise<{ success: boolean; error?: string; data?: LookupEntry }>;
  disabled?: boolean;
  error?: string;
}
```

**Behavior**:
1. Renders a styled `<button>` showing current value (or placeholder if empty)
2. On click: opens a styled absolute `<div>` dropdown listing `items`
3. Clicking an item: selects it, closes dropdown, calls `onChange`
4. At the bottom of the list: "إضافة X" row
5. Clicking "إضافة X": shows an inline mini-form (`<input>` + save/cancel buttons) inside the dropdown
6. On save: calls `onAddNew(name)`, on success: the new entry is selected and dropdown closes; on error: shows error text under the input
7. On cancel: returns to normal list view
8. Click-outside: closes dropdown (uses `useEffect` + `mousedown` listener on document)
9. All Arabic, `dir="rtl"` on root div

### `LookupSection` Component

Settings management panel for one lookup list.

**Props**:
```ts
interface LookupSectionProps {
  title: string;                   // e.g. "أنواع الأصناف"
  items: LookupEntry[];
  loading: boolean;
  addLabel: string;
  onAdd: (name: string) => Promise<void>;
  onEdit: (id: string, name: string) => Promise<void>;
  onDelete: (id: string) => Promise<void>;
}
```

**Renders**: section heading, loading skeleton, list of entries, each row showing: name, predefined badge (if isPredefined), edit button (disabled if isPredefined), delete button (disabled if isPredefined). Inline edit mode per row (click edit → shows input pre-filled, save/cancel). ConfirmDialog on delete.

### IPC Handler Pattern (lookups)

All 12 handlers follow the standard pattern in `electron/main.js`:

```js
ipcMain.handle('lookups:getTypes', () => {
  try {
    const rows = db.prepare(
      `SELECT id, name, is_predefined as isPredefined, is_active as isActive, created_at as createdAt
       FROM item_types WHERE is_active = 1 ORDER BY is_predefined DESC, name ASC`
    ).all();
    return { success: true, data: rows };
  } catch (e) { return { success: false, error: e.message }; }
});

ipcMain.handle('lookups:createType', (_, { name }) => {
  try {
    const trimmed = (name || '').trim();
    if (!trimmed) return { success: false, error: 'الاسم مطلوب' };
    const existing = db.prepare('SELECT id FROM item_types WHERE LOWER(name) = LOWER(?)').get(trimmed);
    if (existing) return { success: false, error: 'هذا النوع موجود مسبقاً' };
    const id = crypto.randomUUID();
    const now = Date.now();
    db.prepare('INSERT INTO item_types (id, name, is_predefined, is_active, created_at, updated_at) VALUES (?,?,0,1,?,?)')
      .run(id, trimmed, now, now);
    const row = db.prepare('SELECT id, name, is_predefined as isPredefined, is_active as isActive, created_at as createdAt FROM item_types WHERE id = ?').get(id);
    return { success: true, data: row };
  } catch (e) { return { success: false, error: e.message }; }
});
// ... similar for updateType (check is_predefined=0), deleteType (set is_active=0)
// ... repeat pattern for color and unit
```
