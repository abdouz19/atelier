# Implementation Plan: Quality Control & Finition

**Branch**: `008-qc-finition` | **Date**: 2026-03-15 | **Spec**: [spec.md](spec.md)

## Summary

Add a Quality Control & Finition module to the Atelier desktop app. The feature introduces a multi-stage post-production pipeline: QC review of returned distribution batches (grading pieces as تالف/مقبول/جيد/جيد جداً), finition operations on accepted pieces, optional chained custom processing steps, and final stock entry when the product is ready. Seven new SQLite tables are added; the feature reads from existing `return_records` and `employees` tables and writes to new `qc_*`, `finition_*`, and `final_stock_entries` tables.

## Technical Context

**Language/Version**: TypeScript 5 strict (frontend renderer) + Node.js plain JS (Electron main process)
**Primary Dependencies**: Next.js 14 App Router, Electron, better-sqlite3, react-hook-form + Zod, Tailwind CSS, Lucide React
**Storage**: SQLite via better-sqlite3 (plain JS in electron/main.js); Drizzle ORM schemas in electron/db/schema/ are reference-only
**Testing**: Manual integration via quickstart.md scenarios
**Target Platform**: Electron desktop app (macOS/Windows)
**Project Type**: Desktop app module addition
**Performance Goals**: All IPC responses < 200ms; KPI query uses indexed aggregations
**Constraints**: Offline-only (SQLite); no external API calls; all UI Arabic RTL
**Scale/Scope**: Single user; hundreds of records per table
