# Implementation Plan: Cutting Parts Size/Color Enhancements

**Branch**: `014-cutting-parts-size-color` | **Date**: 2026-03-22 | **Spec**: [spec.md](./spec.md)

## Summary

Enhance the cutting module to use lookup-based models and parts, record size per session and derive color from fabric. Redesign `cutting_parts` as a cumulative aggregate table keyed by (model, size, color, part). Extend distribution to require size + color selection. Verify/fix final stock filters. See research.md for all decisions.

**Target Platform**: Electron 41 desktop app (macOS primary)
**Project Type**: Desktop application
**Performance Goals**: All IPC responses < 500ms; inventory panel renders < 2s
**Constraints**: No internet required; single-user SQLite; RTL Arabic UI throughout

## Implementation Phases

### Phase 0: DB Migration (electron/main.js)

1. Add `size_label TEXT NOT NULL DEFAULT ''` to `cutting_sessions` (ALTER TABLE ADD COLUMN, idempotent guard)
2. Migrate `cutting_parts` to aggregate schema (rename-copy-drop with `legacy_alter_table = ON`; guard on `SELECT model_name FROM cutting_parts LIMIT 1`)
3. Create `cutting_session_parts` table (`CREATE TABLE IF NOT EXISTS`)

### Phase 1: Backend IPC Handlers (electron/main.js)

1. Update `cutting:create`:
   - Accept `sizeLabel` in payload
   - Insert `size_label` into `cutting_sessions`
   - Insert into `cutting_session_parts` (per-session log)
   - UPSERT into `cutting_parts` using `ON CONFLICT(model_name, size_label, color, part_name) DO UPDATE SET count = count + excluded.count`

2. Update `cutting:getById`:
   - Include `size_label` from `cutting_sessions`
   - Read parts from `cutting_session_parts` (join on session_id)

3. Update `cutting:getAll`:
   - Include `size_label` in each session summary row

4. Update `cutting:getPartSuggestions`:
   - Query `cutting_parts` directly by `model_name` (no session join)

5. Rewrite `cutting:getPartsInventory`:
   - Query `cutting_parts` aggregate + left join distributed quantities
   - Return `sizeLabel` and `color` in each row

6. Add `cutting:getAvailableSizesForModel`:
   - `SELECT DISTINCT size_label FROM cutting_parts WHERE model_name = ? AND count > 0`

7. Add `cutting:getAvailableColorsForModelSize`:
   - `SELECT DISTINCT color FROM cutting_parts WHERE model_name = ? AND size_label = ? AND count > 0`

8. Update `distribution:getAvailablePartsForModel`:
   - Accept `{ modelName, sizeLabel, color }` ‚Äî filter `cutting_parts` by all three

9. Update `distribution:distribute`:
   - Accept `sizeLabel` and `color` in payload
   - Store in `distribution_batches.size_label` and `.color`
   - Availability check uses `cutting_parts` filtered by (model_name, size_label, color, part_name)

10. Verify `final-stock:getRows` applies model/size/color filters (read `electron/features/final-stock/service.js`)

### Phase 2: Frontend Types

1. `frontend/features/cutting/cutting.types.ts`:
   - `CreateCuttingSessionPayload`: add `sizeLabel: string`
   - `CuttingSessionSummary`: add `sizeLabel: string`
   - `CuttingSessionDetail`: add `sizeLabel: string`
   - `PartsInventoryRow`: add `sizeLabel: string`, `color: string`

2. `frontend/features/distribution/distribution.types.ts`:
   - `DistributePayload`: add `sizeLabel: string`, `color: string`

3. `frontend/features/auth/auth.types.ts`:
   - Add `getAvailableSizesForModel`, `getAvailableColorsForModelSize` to `cutting` bridge
   - Update `getAvailablePartsForModel` payload signature

4. `frontend/lib/ipc-client.ts`:
   - Wire new channels + updated signatures

### Phase 3: Frontend Components

1. `CuttingStep1Form.tsx`:
   - Add Size dropdown (sizes from `ipcClient.lookups.getSizes()`)
   - Size is required (add to Zod schema)
   - Export `sizeLabel` in `Step1Values`

2. `NewCuttingSessionModal.tsx`:
   - Pass `sizeLabel: step1Data.sizeLabel` to `ipcClient.cutting.create()`

3. `PartRowsEditor.tsx`:
   - Replace `<input list>` + datalist with a proper `<select>` populated from `ipcClient.lookups.getParts()`
   - Part name is selected from the lookup list (not free text)
   - Keep loading suggestions via `getPartSuggestions` as a secondary hint or remove (parts come from settings now)

4. `CuttingSessionDetail.tsx`:
   - Show `sizeLabel` in the session header

5. `CuttingSessionRow.tsx` / `CuttingSessionTable.tsx`:
   - Show `sizeLabel` column

6. `PartsInventoryPanel.tsx`:
   - Add `sizeLabel` and `color` columns to table
   - Add model/size/color filter dropdowns (local state, not IPC)
   - Group by (model, size, color) with sub-rows per part

7. `DistributeModal.tsx`:
   - After model selection: load available sizes via `getAvailableSizesForModel`
   - After size selection: load available colors via `getAvailableColorsForModelSize`
   - After color selection: load available parts via `getAvailablePartsForModel` with all three filters
   - Pass `sizeLabel` and `color` to `distribute` payload

8. `frontend/public/locales/ar/common.json`:
   - Add new strings for size labels, filter placeholders

### Phase 4: Drizzle Schema (reference only)

1. Create `electron/db/schema/cutting_session_part.ts`
2. Update `electron/db/schema/cutting_part.ts` (new columns, UNIQUE constraint)

## Key Risks

| Risk | Mitigation |
|------|-----------|
| SQLite FK cascade on cutting_parts rename | Use `legacy_alter_table = ON` (same pattern as 013) |
| Legacy sessions with no size_label | Default to `''`; UI shows blank gracefully |
| Distribution modal becomes too complex with 3 dropdowns | Cascade: size unlocks only after model; color only after size |
| final-stock service may not actually filter | Read service.js before implementing; fix if needed |
